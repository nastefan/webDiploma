Setup the Code

Step by step manual how to setup and run the code.

This code contain a lot of branches and setups. Even more, since the analysis code was written by a lot of people, there is a bit mess with what should be set do to run some regular or special job. From my side I've added a few setups, which make my life easier but now it's probaly even more hard to get used for the next code-user. But I will try to explain here all possible setups that I learned and which I've added.

General (must have setup)

1. The "rootlogon.C" file should be placed at the home afs folder:

/afs/..../usename/
will setup "libGutLib" and some other variables which are usefull during the GutCode compilations. You may download it here.

2. Set the path to your main folder in a file which is running when you login into a terminal. In my case it is .zshrc, but it also can be .profile, .bashrc.

You may download here the .zshrc example and use it to update your own setup file. This file should be placed at the same place/level as the "rootlogon.C" file.

Normal regime running

Here I will explain what should be set (by default it will be set when you will load **this** code) to make standard run without hard setups:

1. First check your work-folder:

echo $GUTANADIR

2. Creates mini root files which contain MC and DATA with all possible histograms:

sh $GUTANADIR/run_all_years.sh

3. Next step will create separate folder for the each year and extract histograms in the text format:

sh $GUTANADIR/finilazer_dimuon.sh

4. Go to the created folder, set and run the plotting script, sth. like this:

emacs run_8_plotmaker.C
root -l run_8_plotmaker.C

And here we are, you should get a lot of plots which are produced by run_8_plotmaker.C file with additional libraries. There is the low for file names: run_all_dimuon.sh, finilazer_dimuon.sh etc. in case one would like to try secvtx analysis scripe names will be:

run_all_secvtx.sh
finilazer_secvtx.sh

but the same name for

run_8_plotmaker.C

Special addons


$GUTANADIR/autoscripts/set_srcinc_with_subsv.sh $GUTANADIR/autoscripts/set_srcinc_without_subsets.sh
This part will show addittional part about developing the dimuon dataset collections. To make it possible without breakage old way running this script was developed. This includes additional histograms set, should be used at the very beginning stage + recompile (which it will do automaticaly, just check for the error messages): set_srcinc_with_subsv.sh, set_srcinc_without_subsets.sh while first script will setup all new histograms which are needed for the dimuon subset analysis. second script will get back all changes and make the analysis at the original point.
